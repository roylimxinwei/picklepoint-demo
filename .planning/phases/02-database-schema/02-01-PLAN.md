1. Install supabase CLI as a dev dependency: `npm install supabase --save-dev` (at repo root).

2. Run `npx supabase init` at repo root to create `supabase/config.toml`. If the supabase/ directory already exists from init, proceed.

3. Create the three migration files. The COMPLETE SQL is in 02-RESEARCH.md under "Full Table Schema (Recommended)". Copy that SQL with these specific details:

**Migration 1 — `supabase/migrations/20260305000001_core_tables.sql`:**
All 12 entity tables in dependency order: events, categories, courts, teams, team_members, event_registrations, event_roles, matches, pool_groups, pool_standings, brackets, bracket_slots. Plus deferred FKs from matches to pool_groups/bracket_slots. Plus indexes on event_roles, event_registrations, and matches only.

Key constraints per user decisions:
- events: char(5) event_code UNIQUE, status CHECK (draft/registration/live/completed), created_by references auth.users
- event_roles: UNIQUE(event_id, user_id, role), role CHECK (organizer/referee/player)
- matches: denormalized current_score_team1, current_score_team2, current_game
- event_registrations: UNIQUE(event_id, category_id, user_id)

Do NOT create any index on match_events in this migration — the match_events table does not exist yet and will be created in Migration 2.

**Migration 2 — `supabase/migrations/20260305000002_match_event_sourcing.sql`:**
- match_events table with idempotency_key UUID UNIQUE NOT NULL
- event_type CHECK (point_scored/point_undone), scoring_team CHECK (1/2) nullable
- Snapshot columns: team1_score, team2_score, game_number
- Index on match_events(match_id, created_at) — place here, after the table is created
- sync_match_scores() SECURITY DEFINER trigger function
- AFTER INSERT trigger on match_events

**Migration 3 — `supabase/migrations/20260305000003_rls_and_realtime.sql`:**
- has_event_role(p_event_id, p_role) SECURITY DEFINER STABLE function
- ALTER TABLE ... ENABLE ROW LEVEL SECURITY on ALL 13 tables
- All CREATE POLICY statements per research (anon read for spectator tables, authenticated CRUD with role checks, referee-only match_events insert)
- ALTER PUBLICATION supabase_realtime ADD TABLE public.matches

Important: Use `(select auth.uid())` not bare `auth.uid()` in policy expressions (prevents per-row re-evaluation). Always include TO clause (anon or authenticated) on every policy.

4. Do NOT run `supabase link` or `supabase db push` — that requires user's project ref (checkpoint in next task).
