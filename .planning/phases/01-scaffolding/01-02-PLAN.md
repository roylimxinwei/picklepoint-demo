<task type="auto">
  <name>Task 2: Add Supabase client factories, auth middleware, and API helper</name>
  <files>frontend/src/lib/supabase/client.js, frontend/src/lib/supabase/server.js, frontend/src/middleware.js, frontend/src/lib/api.js, frontend/src/app/api/auth/callback/route.js</files>
  <action>
1. `frontend/src/lib/supabase/client.js` — Export `createClient()` function that calls `createBrowserClient(process.env.NEXT_PUBLIC_SUPABASE_URL, process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY)` from `@supabase/ssr`. This is used in Client Components only.

2. `frontend/src/lib/supabase/server.js` — Export async `createClient()` function that uses `cookies()` from `next/headers` and `createServerClient` from `@supabase/ssr`. Implement cookie `getAll`/`setAll` handlers exactly as shown in the Supabase SSR docs (see RESEARCH.md Pattern 2). Wrap `setAll` in try/catch (Server Components cannot set cookies — only middleware can).

3. `frontend/src/middleware.js` — Place in `src/` directory (NOT in `src/app/`). Create Supabase server client inline (same pattern as server.js but with request/response cookie handling). Call `await supabase.auth.getUser()` to trigger token refresh. Export `config.matcher` that excludes static assets: `['/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)']`.

4. `frontend/src/lib/api.js` — Export `apiFetch(path, options = {})` function. Gets current session via `createClient().auth.getSession()`. Adds `Authorization: Bearer {access_token}` header if session exists. Fetches from `${process.env.NEXT_PUBLIC_API_URL}${path}`. Returns the fetch Response.

5. `frontend/src/app/api/auth/callback/route.js` — OAuth callback route handler. Extracts `code` from URL search params, exchanges it via `supabase.auth.exchangeCodeForSession(code)`, redirects to `/`. This is needed for Google OAuth in Phase 3 but the route should exist now.

CRITICAL: Do NOT put the service role key in any NEXT_PUBLIC_ variable. The anon key is safe to expose (it's rate-limited by RLS). The service role key bypasses RLS and must only be used server-side (FastAPI).
  </action>

  </verify>
  <done>All Supabase client files exist and app builds without errors. middleware.js is in src/ directory. API helper references NEXT_PUBLIC_API_URL. Auth callback route exists.</done>
